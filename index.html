Une vraie vitrine pour montrer un niveau intégrateur / architecte

Ce projet n’est pas une simple présentation : il expose une logique d’exploitation, une supervision web, une simulation temps réel et une architecture crédible pour un recruteur technique. L’objectif est de montrer comment je structure un bâtiment, des lots, des points, des alarmes et des flux de données.

Lecture recruteur

Ce bloc est interactif. Il montre que le site n’est pas juste une plaquette PDF, mais une démonstration de logique de GTB.

Tu peux y expliquer la différence entre une supervision passive et un système exploitable : scénario, consigne, alarmes, historique, retour d’état.

Le mode est volontairement lecture seule pour séparer la démonstration de tout contrôle réel.

Architecture globale expliquée

1. Terrain

CTA, rooftops, éclairage, comptage, capteurs CO₂/température, passerelles LoRaWAN, modules Modbus.

2. Intégration

Middleware pour normaliser les points, publier les états et consommer les événements temps réel.

3. Supervision

Interface web pour vues par zone, alarmes, tendances, énergie, confort et diagnostic d’exploitation.

Ce que j’ai voulu montrer

Je sais structurer un bâtiment en lots techniques, puis l’exposer en objets, scénarios et alarmes.

Je sais séparer la couche de contrôle, la couche d’intégration et la couche de présentation.

Je sais rendre un système compréhensible pour un exploitant, pas seulement pour un automaticien.

WS /ws/live

Base de données logique

Trois familles de tables suffisent pour démarrer proprement :
sites(id, name, location)
zones(id, site_id, name, floor, usage)
points(id, zone_id, protocol, object_name, unit, writable)
alarm_events(id, point_id, severity, label, active, ts)
trends(id, point_id, value, ts)
scenarios(id, name, payload_json)

Comment j’ai conçu le portfolio

  1. J’ai posé un bâtiment crédible, avec lots, zones, usages et logique d’exploitation.
  2. J’ai séparé la vitrine, la démo et la couche de contrôle.
  3. J’ai ajouté une démo live pour illustrer les comportements attendus.
  4. J’ai structuré le site pour qu’un recruteur voie l’architecture autant que le rendu visuel.
